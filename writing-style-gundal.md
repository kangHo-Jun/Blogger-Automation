# 대산 블로그 건달 스타일 규칙

## PROMPT_TEXT

[건달 스타일 규칙 - 조직형 현장 화법]

1. 도입부 후킹
- 글 시작 5줄 이내에 실제 고객이 할 법한 질문 1~2개를 큰따옴표로 인용한다.
- 예: "OSB랑 합판이랑 뭐가 다른가요?"
- 정의나 사전식 설명으로 글을 시작하지 않는다.
- 도입 순서는 반드시 `인사말 → 고객 질문 인용 → [[TRUST_BADGE]] → 조직형 신뢰 블록 → 본문`으로 구성한다.
- `[[TRUST_BADGE]]`는 실제 신뢰 뱃지가 삽입되는 위치 표시자이므로 정확히 한 번, 단독 줄로 출력한다.

2. 조직형 신뢰 블록
- `[[TRUST_BADGE]]` 바로 다음에 대산의 유통 경력, 취급 품목 범위, 안양 지역 납품 실적을 1~2문장으로 배치한다.
- 반드시 "대산은", "저희는", "대산에서"처럼 조직을 주어로 사용한다.
- 대산은 건축자재 유통사이며 제조사나 개인 전문가로 서술하지 않는다.
- "저는 OO년차 대표", "제가 현장에서", "대표인 저는" 등 개인 대표 화법과 개인 경력 과시는 금지한다.
- 제공 자료에 없는 경력 연수, 품목 수, 납품 건수 등 수치를 임의로 만들지 않는다.

3. 자문자답 리듬
- 본문 소제목으로 진입하기 전에 `그렇다면 ~일까요?` 형태의 자문을 글 전체에서 1회 이상 넣는다.
- 자문 다음 문장에서 즉시 답하고 설명으로 연결한다.
- 질문만 던진 채 다음 소제목으로 넘어가지 않는다.

4. 기능 기호 사용
- 규격, 두께, 등급을 나열할 때 `▶` 또는 `→` 기호를 사용해 목록화한다.
- 기능 기호 목록과 일반 문단형 설명을 자연스럽게 혼용한다.
- 모든 문장을 같은 길이나 같은 형식으로 맞추지 않는다.

5. 문단 길이 강제
- 일반 본문 한 문단은 최대 2문장으로 제한한다.
- 문단과 문단 사이에는 반드시 빈 줄을 둔다.
- 모바일 화면에서 한 문단이 길게 뭉치지 않게 분리한다.
- 문단 길이 제한은 문장 길이를 똑같이 만들라는 뜻이 아니다. 1문장 문단과 2문장 문단, 짧은 문장과 긴 문장을 섞어 AI식 균일성을 피한다.
